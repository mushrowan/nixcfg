# nixcfg schema v1

the intermediate representation between language drivers and the nix library.
language drivers emit this JSON, nix consumes it

## top-level

```json
{
  "version": 1,
  "name": "mycel",
  "description": "mycel discord bot configuration",
  "options": { ... }
}
```

| field         | type   | required | notes                              |
|---------------|--------|----------|------------------------------------|
| `version`     | int    | yes      | schema version, currently `1`      |
| `name`        | string | yes      | service/program name               |
| `description` | string | no       | human-readable description         |
| `options`     | object | yes      | keyed by snake_case option name    |

## options

each key in `options` is the option name in snake_case. the nix library
converts to camelCase (`data_dir` → `dataDir`)

```json
{
  "data_dir": {
    "type": "path",
    "default": "/var/lib/mycel",
    "description": "directory for the database, models, and workspace"
  }
}
```

| field         | type         | required | notes                                 |
|---------------|--------------|----------|---------------------------------------|
| `type`        | type         | yes      | see types below                       |
| `description` | string       | no       | from doc comments in source           |
| `default`     | json value   | no       | default value                         |
| `example`     | json value   | no       | example value for docs                |
| `secret`      | bool         | no       | marks as secret, see below            |

## types

### simple types (string shorthand)

| schema type | nix type                  | rust types                          |
|-------------|---------------------------|-------------------------------------|
| `"string"`  | `lib.types.str`           | `String`, `&str`                    |
| `"bool"`    | `lib.types.bool`          | `bool`                              |
| `"int"`     | `lib.types.int`           | `i8`, `i16`, `i32`, `i64`, `isize`  |
| `"uint"`    | `lib.types.ints.unsigned` | `u8`, `u16`, `u32`, `u64`, `usize`  |
| `"path"`    | `lib.types.path`          | `PathBuf`                           |
| `"port"`    | `lib.types.port`          | `u16` with `#[nixcfg(port)]`       |

### complex types (object form)

**optional** — wraps in `nullOr`, defaults to `null` if no explicit default

```json
{ "optional": "string" }
```

→ nix: `lib.types.nullOr lib.types.str`
→ rust: `Option<String>`

**list** — list of a type

```json
{ "list": "string" }
```

→ nix: `lib.types.listOf lib.types.str`
→ rust: `Vec<String>`

**attrs** — attribute set with typed values (keys are always strings)

```json
{ "attrs": "int" }
```

→ nix: `lib.types.attrsOf lib.types.int`
→ rust: `HashMap<String, i32>` or `BTreeMap<String, i32>`

**enum** — one of a fixed set of string values

```json
{ "enum": ["debug", "info", "warn", "error"] }
```

→ nix: `lib.types.enum ["debug" "info" "warn" "error"]`
→ rust: unit-variant enum with `#[derive(NixOptions)]`

**submodule** — nested option group

```json
{
  "submodule": {
    "host": { "type": "string", "default": "localhost" },
    "port": { "type": "port", "default": 5432 }
  }
}
```

→ nix: `lib.types.submodule { options = { host = lib.mkOption { ... }; ... }; }`
→ rust: nested struct with `#[derive(NixOptions)]`

### type composition

types compose freely:

| schema                                    | nix type                                        | rust type                  |
|-------------------------------------------|-------------------------------------------------|----------------------------|
| `{ "optional": { "list": "string" } }`   | `nullOr (listOf str)`                           | `Option<Vec<String>>`      |
| `{ "list": { "attrs": "int" } }`         | `listOf (attrsOf int)`                          | `Vec<HashMap<String, i32>>`|
| `{ "optional": { "submodule": {...} } }`  | `nullOr (submodule {...})`                      | `Option<NestedStruct>`     |

## secrets

when `secret: true` is set on an option:

1. the nix option name gets a `Path` suffix (`discord_token` → `discordTokenPath`)
2. the nix type becomes `path` (or `nullOr path` if the original type was optional)
3. the description is prefixed with "path to file containing "
4. any `default` on the original option is dropped (secrets don't have defaults,
   though optional secrets default to `null`)

the schema preserves the original type so language drivers know what the
actual value type is. the nix library handles the file-path transformation

### collision avoidance

since `#[nixcfg(secret)]` on a field named `foo` generates an option called
`foo_path`, any sibling field named `foo_path` or `foo_file` is automatically
excluded from the schema to avoid a collision. this means structs like:

```rust
struct Config {
    #[nixcfg(secret)]
    api_token: String,         // → generates api_token_path option
    api_token_path: Option<PathBuf>,  // auto-skipped (would collide)
}
```

produce a single `api_token_path` option in the schema, not two. this is
intentional: many programs already have a `_path` field for loading secrets
from files, and `#[nixcfg(secret)]` replaces it with the nixcfg convention.

if a `_path`/`_file` field is NOT a sibling of a secret, it is included
normally. only exact matches of `{secret_name}_path` and
`{secret_name}_file` are skipped

language drivers should generate corresponding `--foo-path` CLI args that
read the value from the file at runtime

## naming conventions

- schema uses **snake_case** for option names (language-neutral)
- nix library converts to **camelCase** (`data_dir` → `dataDir`)
- CLI flags use **kebab-case** (`data_dir` → `--data-dir`)
- environment variables use **SCREAMING_SNAKE_CASE** (`data_dir` → `DATA_DIR`)
- language drivers should use the source language's convention and convert
  to snake_case when emitting the schema

## config format helpers

the nix library provides helpers to convert evaluated module config into
formats suitable for passing to the service at runtime

### CLI args (`toCliArgs schema cfg`)

returns a list of strings for use with `lib.escapeShellArgs`

| type    | value   | result                              |
|---------|---------|-------------------------------------|
| string  | `"foo"` | `["--flag" "foo"]`                  |
| bool    | `true`  | `["--flag"]`                        |
| bool    | `false` | `[]` (omitted)                      |
| list    | `[1 2]` | `["--flag" "1" "--flag" "2"]`       |
| null    | -       | `[]` (omitted)                      |
| secret  | path    | `["--name-file" "/path"]`           |

### environment variables (`toEnvVars schema cfg`)

returns an attrset of `NAME = "value"` pairs

| type    | value   | result                              |
|---------|---------|-------------------------------------|
| string  | `"foo"` | `{ FLAG = "foo"; }`                 |
| bool    | `true`  | `{ FLAG = "true"; }`               |
| bool    | `false` | `{ FLAG = "false"; }`              |
| list    | `[1 2]` | `{ FLAG = "1,2"; }`               |
| null    | -       | omitted                             |
| secret  | path    | `{ NAME_FILE = "/path"; }`         |

### config attrset (`toConfigAttrs schema cfg`)

returns a nix attrset with snake_case keys, suitable for serialising to
JSON/TOML/YAML as a config file. null values are omitted. secrets appear
as `name_path` keys
